1 Overview

HuatuoGPT (华佗GPT) is an open-source family of Chinese medical large language models developed by the FreedomIntelligence group at The Chinese University of Hong Kong, Shenzhen (CUHKSZ) and the Shenzhen Research Institute of Big Data (SRIBD). It was named after Hua Tuo, the legendary Chinese physician from the Eastern Han dynasty.

The project's stated goal is "Towards Taming Language Models To Be a Doctor" — building a professional medical consultation AI for Chinese-speaking populations. Unlike Google's Med-PaLM (closed, proprietary), HuatuoGPT is fully open-source under Apache-2.0 and has achieved real hospital deployment at a scale few medical AI projects can match.

3 Architecture & Training

3.1 Data Strategy (v1 — the core innovation)

HuatuoGPT's key insight is using both ChatGPT-distilled data and real doctor-patient data, then aligning the model through Reinforcement Learning from Mixed Feedback (RLMF):

Data TypeSourceSizePurpose
Distilled InstructionsChatGPT self-instruct61,400Broad medical knowledge coverage
Real Doctor InstructionsReal consultation records69,768Single-round real clinical QA
Distilled ConversationsTwo ChatGPTs role-playing68,888Multi-turn dialogue practice
Real Doctor ConversationsReal doctor-patient records25,986Authentic diagnostic dialogue patterns

Total SFT data: ~226,042 examples. Released publicly on HuggingFace.

3.2 RLMF — Reinforcement Learning from Mixed Feedback

The reward model is trained on both ChatGPT outputs and real doctor consultations, learning the merits of each. The LLM is then aligned through PPO to prefer responses that combine ChatGPT's knowledge breadth with real doctors' interactive diagnostic quality. This is HuatuoGPT's primary research contribution.

3.3 HuatuoGPT-o1 — "Slow Thinking" for Medicine

The o1 variant introduces a two-stage reasoning pipeline:

  1. SFT Stage: Trained on 40,000 verifiable medical problems. A GPT-4o-powered verifier (96.5% accuracy) guides search for complex reasoning trajectories (complex chain-of-thought).
  2. RL Stage (PPO): A dedicated medical_o1_verifier_3B model provides sparse rewards to refine reasoning accuracy.

Output format: ## Thinking [detailed reasoning] ## Final Response [concise answer]

4 Benchmarks & Evaluation

4.1 Original HuatuoGPT Evaluation

The paper evaluated against a T5-finetuned baseline on three Chinese medical datasets. The key finding: HuatuoGPT achieves much higher response diversity (Distinct-1/2: 0.74/0.93 vs T5's 0.41/0.52) at the cost of lower n-gram overlap (BLEU/ROUGE):

Pattern: HuatuoGPT's o1 variant leads among small open-source models. But across the board, general-purpose LLMs (GPT-4, ERNIE Bot, Tongyi Qianwen) consistently outperform HuatuoGPT on standardized Western medicine benchmarks. Its real strength is in Chinese-language triage and TCM, not general medical QA.

4.3 Response Diversity vs. Accuracy Trade-off

The v1 paper's own data reveals a fundamental design choice: HuatuoGPT prioritizes diverse, natural-sounding responses over template-like accuracy. Its Distinct scores (0.74/0.93) far exceed the T5 baseline (0.41/0.52), but at the cost of lower BLEU/ROUGE. This is appropriate for conversational triage but problematic for clinical decision support where precision matters.

8 How to Try HuatuoGPT

There is no official public API for HuatuoGPT. The original demo site (huatuogpt.cn) appears inactive. However, all model weights are open-source and can be run locally or deployed as a self-hosted API. Here are the available options:

8.1 Option 1: Direct Inference via HuggingFace Transformers (Easiest)

Run the o1-7B (Chinese + English) or o1-8B (English) model locally with a few lines of Python:

Hardware Note: 7B/8B models require ~16GB GPU RAM (consumer GPU like RTX 3090/4090). 70B/72B models need multi-GPU setups. All are Apache-2.0 licensed.
ModelBackboneLanguageHuggingFace
HuatuoGPT-o1-7BQwen2.5-7BEnglish + ChineseFreedomIntelligence/HuatuoGPT-o1-7B
HuatuoGPT-o1-8BLLaMA 3.1-8BEnglishFreedomIntelligence/HuatuoGPT-o1-8B
HuatuoGPT-o1-70BLLaMA 3.1-70BEnglishFreedomIntelligence/HuatuoGPT-o1-70B
HuatuoGPT-o1-72BQwen2.5-72BEnglish + ChineseFreedomIntelligence/HuatuoGPT-o1-72B
HuatuoGPT-Vision-7BQwen2.5-VLMultimodalVision model on HF

Inference code (o1-7B for Chinese):

from transformers import AutoModelForCausalLM, AutoTokenizer

model = AutoModelForCausalLM.from_pretrained(
    "FreedomIntelligence/HuatuoGPT-o1-7B",
    torch_dtype="auto",
    device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained(
    "FreedomIntelligence/HuatuoGPT-o1-7B"
)

input_text = "我最近总是咳嗽，怎么办？"
messages = [{"role": "user", "content": input_text}]

inputs = tokenizer(
    tokenizer.apply_chat_template(
        messages, tokenize=False, add_generation_prompt=True
    ),
    return_tensors="pt"
).to(model.device)

outputs = model.generate(**inputs, max_new_tokens=2048)
print(tokenizer.decode(outputs[0], skip_special_tokens=True))
# Output format:
# ## Thinking [reasoning process]
# ## Final Response [answer]

8.2 Option 2: Self-Hosted API via vLLM or SGLang (Production)

For API-style access, deploy with SGLang (recommended by the authors) which provides an OpenAI-compatible endpoint:

# Start SGLang server with HuatuoGPT-o1-8B
python -m sglang.launch_server \
    --model-path FreedomIntelligence/HuatuoGPT-o1-8B \
    --port 2835 \
    --mem-fraction-static 0.8

# Then query via OpenAI-compatible API:
curl http://localhost:2835/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "FreedomIntelligence/HuatuoGPT-o1-8B",
    "messages": [{"role": "user", "content": "How to stop a cough?"}]
  }'

8.3 Option 3: Original HuatuoGPT v1 (Research Version)

The original HuatuoGPT (v1, EMNLP 2023) is also open source:

  • GitHub: FreedomIntelligence/HuatuoGPT
  • HuatuoGPT-7B (Baichuan-7B-based): Full weights on HuggingFace
  • HuatuoGPT-13B (LLaMA-based): Delta weights only; requires running apply_delta.py with original LLaMA weights
  • CLI demo: pip install -r requirements.txt && python huatuo_cli_demo_stream.py --model-name $model_dir
  • SFT data: HuatuoGPT-sft-data-v1 (~226K examples, Apache-2.0)
No Free Hosted API: There is no ChatGPT-style website or API key to try HuatuoGPT instantly. Unlike Med-PaLM (available as MedLM on Google Cloud), HuatuoGPT requires self-hosting. The trade-off is full control, no per-token cost, and Apache-2.0 license freedom. For a quick test, use the HuggingFace transformers snippet above — it runs on any GPU with 16GB+ VRAM.

9.6 Chinese Cloud Providers (If Hosting in China)

Chinese GPU prices are currently in an upcycle (+5-40% hikes in 2026) due to AI demand and hardware shortages. H100 lease rates rose from ~$1.70/hr to ~$2.35/hr over 6 months. Key providers:

  • Alibaba Cloud (Aliyun): Pingtouge compute cards; prices recently +5-34%. Best availability in China.
  • Tencent Cloud: Hunyuan model API pricing surged ~460%; raw GPU compute also +5%. Strong inference infrastructure.
  • Baidu Cloud: AI compute +5-30% hike. Integrated with ERNIE ecosystem.
  • Domestic GPU (Huawei Ascend, etc.): ~80,000 RMB/month per card — more expensive than international options but no export restriction risk.

9.7 Recommendation

For the 7B model (recommended starting point): Use RunPod RTX 4090 ($0.44/hr, ~$320/mo) — it's the best price-to-reliability ratio for a Chinese medical chatbot. Deploy with vLLM (one-click template available) for OpenAI-compatible API. If traffic is sporadic, use serverless (RunPod serverless or Modal) to pay only for actual inference time, bringing costs down to $50-150/mo for light usage.
For the 72B model: Budget $2,500-4,000/mo for 2× A100. Consider quantizing to INT4 to run on a single A100 80GB ($1.10-1.79/hr, ~$800-1,300/mo) instead. The quality difference is usually negligible for Chinese medical triage.
